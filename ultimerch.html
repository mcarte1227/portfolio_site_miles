UltiMerch

Responsive E-commerce web app where gaming fans can purchase a variety of official merchandise from various franchises and companies.

SKILLS USED

  • User research
  • User stories + flow diagrams
  • Usability testing
  • Low + Mid fidelity wireframes
  • Branding guidelines
  • User Testing
  • User interface design
  • Reaponsive layout design

INTRODUCTION

UltiMerch was created for gamers looking to buy various forms of merchandise from their favorite series and/or companies, all on one collective web application.

APPROACH

The project brief called for me to design a responsive and/or mobile e-commerce app where users can search for and buy products efficiently to meet their specific needs. This app is targeted for people who are unable to find the items they would in traditional retail stores. Key features to be included are methods of filtering inventory and delivery to any address, with a free return policy being a plus.

USER STORIES + FLOW DIAGRAM

I chose three user story statements from the brief to complete user flow diagrams, one which I would choose to base the layout of the app itself on.

  • “As a frequent customer, I want to be able to save my favorite search criteria, so that I can easily find what is most relevant to me.”
  • “As a customer, I want to be able to place multiple items in a shopping cart, so that I can purchase more than one item at a time.”
  • “As a returning customer, I want to see an overview of my previous activity, so that I can view previous purchases and search history.”

INITIAL SKETCHES

I then proceeded to create quick, low fidelity sketches for the e-commerce project based on the user story quote I have chosen.

BRANDING

In order to create a cohesive brand and identity, I took some time to draw influence for the look of the site. The various shades of blue used for the color palette, the shape of the image boxes, buttons and other elements, were done in an attempt to fit the “futuristic” image of gaming I had in my mind throughout this project.

WIREFRAMES

After the initial sketching phase, it was time for the creation of the mid-fidelity wireframes showcased below. After which I created an interactive prototype for a select number of individuals to test the navigation and flow of the app.

USER TESTING

Before applying the final look to the app, I created an interactive prototype in InVision to test how the product itself flowed and sent it to three users with the simple task of choosing a product from the home page, adding two variations of the same item to the shopping cart, and proceeding to check out.

CONCLUSION

Overall, this project was simple, open-ended, and enjoyable for me to work on. I tried to take all of the design patterns and techniques I had learned through the course to create the most professional-looking project I had done up to this point. While I can say that I improved on my visual designing skills, I still have a bit of improving to do when it comes to the communication-based aspect of the UX / UI design process.
